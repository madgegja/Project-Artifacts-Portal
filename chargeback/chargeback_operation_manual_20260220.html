3. 소명 자료 작성 (공통 — 전 PG사 동일)

⚠️ 필수 권한: 투믹스글로벌 관리자 — 회원 정보, 결제 내역 발송 관리 권한 필요

3-1. 소명 자료 다운로드

  1. PG사 차지백 사이트에서 해당 회원의 거래 번호(Transaction Number) 복사
  2. 투믹스 글로벌 관리자 > Payments > 결제 내역 발송 관리 이동
  3. Period를 결제일로 설정 (해당 차지백 거래의 결제 날짜 기준)
  4. Search Terms에서 Transaction Number 선택 후 거래번호 붙여넣기 → 검색
  5. 결제 내역 선택 → Receipt 캡처 후 저장
    파일명: Receipt_[거래번호]
  6. 회원 Info에서 자동결제 중이면 → 관리자가 수동으로 자동결제 해지
  7. 회원 Info > "차지백 소명자료" 버튼 클릭 > Chargeback explanation → 기간 설정 → Download 엑셀 파일 다운로드

3-2. 소명 자료 작성 (표준 템플릿 사용)

  1. Chargeback_Explanation_Form_v5.xlsx 열기 → 다른 이름으로 저장 (원본 보존)
  2. STEP 1 — 다운로드한 소명자료 엑셀 전체를 데이터 영역에 한 번에 붙여넣기
    ※ 붙여넣기 전 확인: 차지백 접수된 거래내역 외 삭제 (타 PG사 결제 포함), Accessed URLs · 1:1 history 삭제
  3. STEP 2 — Proof of Delivery — 해당 Case(A~E) 1개만 남기고 나머지 행 삭제 → [대괄호] 부분 입력
  4. STEP 3 — Invoice — VIP / Coin 중 해당 답변만 남기고 삭제 → [대괄호] 부분 입력
  5. STEP 4 — Receipt 스크린샷 삽입
  6. User Account > User id가 "숫자+"로 표시되면 → 셀 서식을 '숫자'로 변경

3-3. PDF 변환

  1. 소명 자료 작성 완료 후 PDF 파일로 변환하여 저장
  2. 파일명 규칙: [PG사] Chargeback_[거래번호].pdf
    예: CKO Chargeback_pay_abc123.pdf, Paymentwall Chargeback_TXN456.pdf

Q: Airwallex에서 분쟁 격상되면?→ 분쟁 금액 재공제 + USD $15 중재 수수료 부과
Q: PayPal 인출 한도 초과하면?→ 금액이 되돌아옴. 일 USD 20,000 한도 준수
Q: Paymentwall 답변은 왜 안 쓰나?→ Paymentwall은 소명 자료(PDF)만 제출하는 구조
Q: 제재 기준은?→ Fraudulent(사기): 즉시 제재 / 그 외: 의심 처리 후 판단
Q: WorldPay 결제 검증 오류 시?→ 대시보드 결제완료 ↔ 관리자 결제시도 불일치는 개발팀 에스컬레이션
Q: 소규모 PG 차지백 대응은?→ WorldPay/Payletter/OPN은 동일한 소명 자료 + PDF 제출 프로세스 적용. MyCard는 예외 (이메일 기반, MyCard 지정 양식)
Q: MyCard 14일 기한 초과 시?→ 이용내역 유무와 관계없이 환불 거부. MyCard 양식에 "환불 불가 (14일 기한 초과)" 기입 후 회신
