AI 챗봇이란? — 운영 개요

한 줄 요약

고객이 웹사이트/앱에서 AI와 직접 대화하여 간단한 CS 문의(구독 해지, FAQ 등)를 셀프서비스로 안내하고, 복잡한 건은 상담원에게 자동 전달되는 시스템입니다.

핵심 원칙

AI 챗봇은 "안내만 하고, 실행은 하지 않습니다." 구독 해지 '방법'을 알려주지만, 직접 해지하지 않습니다. 환불 '절차'를 안내하지만, 환불을 승인하지 않습니다.

상담원이 집중해야 할 업무

AI 도입 후 상담원은 단순 반복 업무에서 해방되고, 아래 업무에 집중하게 됩니다:

  1. 환불 적격성 판단 — 정책 기준에 따라 환불 가능 여부 결정
  2. 복잡한 계정 문제 — 소셜 로그인 복구, 계정 병합, GDPR 삭제 요청
  3. 결제 분쟁 — 이중 과금, 무단 결제 확인 및 처리
  4. AI 에스컬레이션 건 검토 — AI가 작성한 초안을 검토/수정 후 발송
  5. Content 관련 문의 — 작품 오류, 번역 이슈 등 전체 직접 처리
  6. Fraud 의심 건 최종 판단 — AI가 플래그한 건의 실제 사기 여부 판단

L2 "AI 초안" 이란?

AI가 고객 문의를 분석하고 응답 초안을 작성해 줍니다. 상담원은 이 초안을 반드시 검토한 후, 필요시 수정하여 고객에게 발송합니다. AI 초안을 그대로 보내도 되지만, 정책 판단이 필요한 부분은 꼭 확인하세요.

L2 에스컬레이션 처리 절차

AI 초안 검토 체크리스트

  • 고객 문의 내용과 초안 응답이 일치하는지 확인
  • 환불/보상 관련 내용이 있다면 정책 기준에 맞는지 확인
  • 금액, 날짜, 기한 등 구체적 수치가 정확한지 확인
  • 고객 감정에 맞는 톤인지 확인 (화난 고객 → 공감 표현 보강)
  • 마켓에 맞는 언어/URL이 사용되었는지 확인
  • 문제없으면 발송, 수정 필요시 수정 후 발송

L1 에스컬레이션 처리 절차

L1은 AI가 응답을 전혀 하지 않은 건입니다. 상담원이 처음부터 직접 응대합니다.

L1 즉시 확인 사항

  • Fraud 점수가 높은 경우 → 위험 신호 목록 먼저 확인
  • 법적/컴플라이언스 문의 → 관련 정책 확인 후 응대
  • Content/Other 카테고리 → 해당 팀에 전달 또는 직접 처리
  • 미지원 언어 → 해당 마켓 상담원에게 재배정

사기(Fraud) 의심 대응

AI가 감지하는 위험 신호

AI가 자동으로 9가지 위험 신호를 체크합니다. 위험도에 따라 두 그룹으로 나뉩니다.

Critical 지표 (1개라도 → 주의)

#위험 신호예시
1타인 명의 계정 접근 시도"남편/여자친구 계정 해지해주세요"
2무단 결제 + 즉시 환불 요구"모르는 결제가 있어요, 당장 환불해주세요"
3차지백/은행 분쟁 위협"환불 안 해주면 은행에 분쟁 걸겠습니다"
4미성년자 결제 주장"아이가 실수로 결제했어요"
5계정 탈취 의심 패턴비밀번호 변경 + 이메일 변경 동시 요청

Supporting 지표 (2개 이상 조합 → 주의)

#위험 신호예시
6시간 압박 / 긴급성 강조"오늘 안에 처리해주세요, 급합니다"
7다중 환불 요청같은 계정에서 반복적 환불 요구
8정보 불일치이름, 이메일, 결제 정보가 일관되지 않음
9소셜 엔지니어링 시도상담원 감정에 호소하며 정책 예외 요구

Fraud 의심 시 상담원 대응 절차

Fraud 플래그 확인 시 행동 지침

  • AI가 전달한 Fraud 점수와 감지된 신호 목록 확인
  • 고객에게 추가 본인 확인 정보 요청 (결제 정보, 가입 이메일 등)
  • 절대 Fraud를 직접 언급하지 않기 ("보안을 위해 확인이 필요합니다")
  • 고객 응대를 최우선 — 차지백 언급 자체만으로 거부하지 않기
  • 확실한 사기 판단 시 → 팀 리더에게 보고

절대 하지 말 것

고객에게 "사기 의심", "Fraud", "부정 이용" 등의 단어를 사용하지 마세요. 항상 중립적이고 정중한 표현으로 본인 확인을 요청하세요.

자주 묻는 질문 (FAQ)

Q. AI가 잘못된 답변을 보낸 것 같아요. 어떻게 하나요?

L3 자동 응답 건에서 오류를 발견하면, 해당 건을 캡처하여 팀 리더에게 보고해주세요. 빠르게 템플릿을 수정합니다. AI 자동 응답은 사전 검증된 템플릿 기반이므로 오류가 드물지만, 발견 시 즉시 보고가 중요합니다.

Q. 고객이 "AI와 대화하고 싶지 않다, 사람과 연결해달라"고 하면?

AI 챗봇에는 항상 "CS 티켓 제출" 옵션이 함께 제공됩니다. 고객이 사람과의 대화를 원하면 티켓이 생성되고 상담원에게 전달됩니다. 이 경우 평소처럼 직접 응대하면 됩니다.

Q. 이메일로 들어온 문의도 AI가 처리하나요?

아니요. 이메일 접수 건(전체의 약 42%)은 기존과 동일하게 상담원이 직접 처리합니다. AI 챗봇은 웹사이트/앱의 실시간 채팅 채널에서만 작동합니다.

Q. AI가 분류한 카테고리가 틀린 것 같아요.

AI 분류 정확도는 약 92% 이상이지만, 복합 의도(예: 해지 + 환불 동시 요청)에서 간혹 오분류가 발생할 수 있습니다. 틀린 분류를 발견하면 해당 건을 기록해주세요 — 피드백이 축적되면 AI 분류 정확도가 개선됩니다.

Q. AI 챗봇이 다운되면 어떻게 되나요?

챗봇 서비스 장애 시 고객 문의는 자동으로 이메일 채널로 전환됩니다. 상담원은 기존 이메일 처리 방식으로 정상 운영하면 됩니다. 시스템 복구는 개발팀에서 처리합니다.

Q. 고객이 AI 응답에 불만을 표시하면?

고객의 불만 피드백은 자동 수집됩니다. 상담원이 추가 대응이 필요하다고 판단되면 해당 건을 직접 이어받아 처리하세요. "AI 답변이 부정확했다면 사과드립니다" 등의 표현은 자연스럽게 사용 가능합니다.

  • L1 긴급 건 우선 확인 — Fraud 플래그 건 먼저
  • L2 AI 초안 대기 건 확인 — 검토 후 순차 발송
  • 이메일 채널 미답변 건 확인

업무 중

상시 확인 사항

  • 새로운 에스컬레이션 알림 수시 확인
  • L2 AI 초안 검토 시 → 금액/날짜/URL 정확성 반드시 확인
  • 고객 감정이 격앙된 건 → 공감 표현 보강 후 발송
  • Fraud 플래그 건 → 추가 확인 없이 환불/해지 처리하지 않기
  • 분류 오류 발견 시 기록

  • 미처리 L1 건 유무 확인 — 있으면 다음 교대 인수인계
  • 대기 중인 L2 초안 건 처리 완료 확인
  • 당일 발견한 AI 오류/개선점 기록 정리

챗봇 서비스 장애 (응답 불가)

영향: 챗봇 채널 고객이 응답 받지 못함
조치: 이메일 채널로 자동 전환됨. 상담원은 이메일 문의량 증가에 대비. 개발팀 장애 복구 진행 중일 것이므로 별도 신고 불필요 (자동 알림 체계).

AI가 부적절한 응답 반복 발송

영향: 고객 불만 증가
조치: 즉시 팀 리더에게 보고. 해당 카테고리/Sub-Intent 파악하여 전달. 개발팀에서 해당 템플릿 긴급 수정 또는 임시 비활성화 가능.

대량 Fraud 의심 건 동시 접수

영향: 조직적 사기 가능성
조치: 패턴 파악 (동일 IP, 유사 문구, 동시간대) → 팀 리더 + 보안팀 즉시 보고. 개별 건은 정상 Fraud 절차 따라 처리.

특정 마켓에서 문의 급증

영향: 해당 마켓 응답 지연
조치: 결제 시스템 장애, 콘텐츠 이슈 등 원인 파악. 공통 이슈라면 해당 마켓 일괄 안내 템플릿 요청.
