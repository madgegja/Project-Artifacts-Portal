🔌 라우팅 결정 (L1 / L2 / L3)

🔴 L1 -- Immediate Escalation

AI 응답 없음. 즉시 티켓 제출.

  • Fraud HIGH (50점 이상)
  • Content 카테고리 전체
  • Other 카테고리 전체
  • 법적 키워드 감지 (변호사, GDPR 등)
  • 차지백/은행 분쟁 키워드
  • 버그 리포트 (기기명+OS 버전)
  • 무단결제, 이중결제, 미성년자
  • 계정 잠금/제한
  • 욕설, 극단적 감정 표현

🟠 L2 -- AI Draft + Agent Review

AI 초안 생성, 상담원 검토 후 발송.

  • 환불 적격성 판단 필요
  • 해지 + 환불 동시 요청
  • 자동갱신 미인지 불만
  • 소셜 로그인 복구
  • 이메일 변경
  • 결제 후 미적용
  • 코인 이슈
  • 동일 이슈 2회+ 반복 접수

🟢 L3 -- Self-Service Auto Response

검증된 안내 자동 발송. 티켓 옵션 포함.

  • 구독 해지 안내
  • 갱신일/잔여기간 문의
  • VIP 접근 불가 안내
  • 도메인 혼동 안내
  • iOS 환불 리디렉트
  • 환불 상태 안내
  • 비밀번호 리셋
  • 계정 삭제 체크리스트
  • 결제 수단/실패 안내
  • 일반 기술 지원 / 앱 이슈
⚠️
우선순위 규칙: L1 > L2 > L3. L1 트리거가 하나라도 있으면 다른 레벨 무시. Fraud HIGH는 무조건 L1. 복합 의도 3개 이상 또는 의도 분류 불가(confidence < 0.50)도 L1.

Fraud 탐지 운영

🚩 Fraud 점수 체계

모든 고객 메시지에 대해 실시간으로 Fraud 탐지를 수행합니다. 9개 Red Flag 지표(Critical 5 + Supporting 4)로 구성됩니다.

LOW (0~19)
MEDIUM (20~49)
HIGH (50~100) → 강제 L1
점수 범위위험 등급동작
0 ~ 19 ⚫ LOW 정상 진행. 스킬 2(셀프서비스) 또는 3(에스컬레이션)으로 라우팅.
20 ~ 49 ⚫ MEDIUM 주의 플래그 기록. 정상 처리하되 메타데이터에 플래그 기록.
50 ~ 100 ⚫ HIGH 강제 L1 에스컬레이션. Fraud 플래그 + 맥락 요약 상담원 전달.

🔴 Critical 지표 (5개) -- 1개라도 감지 시 주의

Critical 지표 1개 단독 감지 시 해당 점수 반영. 점수 합계 50+ 시 L1 에스컬레이션.

#Red Flag한국어명점수설명
C1 content_viewed_unauthorized 콘텐츠 열람 후 무단거래 주장 +40 "unauthorized" 주장 + 계정에 결제 후 VIP 열람 이력 존재. 백엔드 검증 필요.
C2 ai_legal_document AI 생성 법률 문서 패턴 +35 과도한 법률 용어, EU 지침 번호 인용, 7일 데드라인, 포맷화된 클레임 레터 패턴.
C3 fake_legal_rep 가짜 법률 대리인 사칭 +35 "on behalf of my client" + 검증 불가 법무법인명 + Gmail 등 무료 이메일 도메인.
C4 contradictory_claims 주장 간 모순 +30 대화 중 날짜, 금액, 상품, 상황 설명이 메시지마다 변경되는 패턴.
C5 strategic_escalation 전략적 에스컬레이션 패턴 +30 정중한 요청 → 권리 주장 → 법적 위협 → 대리인 동원으로 단계적 위협 수위 상승.

🟠 Supporting 지표 (4개) -- 2개 이상 조합 시 주의

Supporting 지표 단독으로는 에스컬레이션하지 않음. 2개+ 조합 시 합산 점수 반영.

#Red Flag한국어명점수설명
S6 fake_name 가짜/밈 이름 사용 +15 명백히 허구인 이름 (만화 캐릭터, 밈, 숫자 포함 이름 등). 문화권별 차이 고려.
S7 unverifiable_info 검증 불가 정보 +15 존재하지 않는 기관명, 잘못된 법률 조항 인용, 허위 참조 번호.
S8 exchange_rate_claim 환율 손해 주장 +15 PG사/카드사 환전 수수료를 투믹스 책임으로 전가하여 추가 보상 요구.
S9 chargeback_threat_repeat 차지백 위협 반복 +15 대화 중 차지백/은행 분쟁을 2회 이상 반복 언급. 참고 수준, 고객 응대 우선.

📜 Fraud 탐지 핵심 원칙

원칙설명
탐지만 수행어떤 경우에도 자동 차단/거부하지 않음. 플래그 기록 + 상담원 전달만.
고객 비노출Fraud 점수, Red Flag 정보를 고객에게 절대 보여주지 않음.
의심 ≠ 확정"의심된다"로 표현. "사기다"라고 단정하지 않음.
대화 맥락 누적단일 메시지가 아닌 대화 전체에서 패턴 감지.
선의 추정동일 증거에 합리적 해석이 가능하면 고객에게 유리하게 판단.

자동화 영구 배제 영역

🚫 어떤 Phase에서도 자동 응답을 하지 않는 영역

아래 항목은 시스템 고도화와 무관하게 영구적으로 자동 처리에서 배제됩니다.

  • ✖
    Content 카테고리 전체
    작품 업데이트 일정, 번역/맞춤법 에러, 챕터 누락, 이미지 깨짐, 작품 투고/제안, 작품 피드백/평점. 인간 판단 필수.
  • ✖
    Other 카테고리 전체
    구직, 파트너십, 비즈니스 문의, CS 대상 아닌 문의. 적절한 부서 배정 필요.
  • ✖
    기술 버그 리포트 (기기+OS)
    기기명 + OS 버전이 포함된 상세 버그 리포트. 개발팀 전달 필요. (예: "iPhone 15 Pro, iOS 17.2에서 크래시 발생")
  • ✖
    법적/컴플라이언스 문의
    EU 철회권(Widerrufsrecht, Droit de rétractation), GDPR(DSGVO, RGPD), CDC 등 법적 규정 인용. 법적 판단 불가.
  • ✖
    Fraud HIGH (50점 이상)
    Red Flag Critical 1개 이상 또는 Supporting 2개 이상 조합으로 fraud_score 50+ 달성. 인간 검토 필수.
  • ✖
    제3자 해지 요청 (Minna Technologies / Ethoca)
    @cancel.us.minnatechnologies.com 도메인 발신 자동 요청. 절대 응답 금지 (DO NOT RESPOND).

품질 규칙

📐 응대 품질 필수 규칙

항목규칙상세
도메인 URL 형식 toomics.com/en, toomics.com/fr 형태 사용 en.toomics.com 형식은 사용하지 않음 -- 실제 상담원 데이터와 불일치. 일본: toomics.jp/ja/, Family Safe: toomics.site
날짜/시간 표기 모든 날짜/시간은 KST (한국 표준시) 예: "2026-02-22 KST". 실제 상담원 응답의 100%가 KST 사용.
차지백 관련 방침 고객 응대를 최우선으로 처리 차지백 단독으로 L1 강제 에스컬레이션하지 않음 (Fraud 점수에만 반영). 고객 문제 해결에 집중.
티켓 제출 옵션 모든 자동 응답에 필수 포함 "If you need further assistance, you can submit a support ticket." (또는 해당 언어 번역)
응답 길이 최대 300단어 이내 벽 텍스트(wall of text) 방지. 7단계 이상 시 분할.
공감 표현 응답 첫 줄에 공감 한 문장 필수 "I understand..." / "Je comprends..." 등으로 시작
언어 일관성 감지된 언어로 전체 응답 작성 언어 혼용 금지. 최초 감지 언어로 대화 전체 유지.
대화 턴 제한 최대 5턴 5턴 초과 시 에스컬레이션 제안.
확정 약속 금지 "확인 후 안내" 표현만 사용 환불 승인, 처리 완료 등 확정적 약속 표현 절대 금지.
